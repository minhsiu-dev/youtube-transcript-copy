# YT Transcript Copier

A Chrome extension (Manifest V3) that copies the transcript of the YouTube
video you're currently watching to your clipboard.

## Features

- One-click copy from the toolbar icon
- Three output formats: plain text, with timestamps, plain + title/URL
- Multi-language: defaults to the original-language transcript, switchable
  to any available language (including auto-translated)
- Icon reflects state: red when a transcript is available, gray otherwise

## Install (development)

```bash
npm install
npm run icons   # one-time: generate icon PNGs
npm run build
```

Then in Chrome:

1. Visit `chrome://extensions`
2. Toggle **Developer mode** (top-right)
3. Click **Load unpacked** and choose the `dist/` folder

For iterative development, run `npm run watch` and reload the extension
from `chrome://extensions` after each change.

## How it works

YouTube embeds caption metadata (language list + URLs) in
`window.ytInitialPlayerResponse`. A MAIN-world content script reads it.
The underlying `/api/timedtext` endpoint requires a `pot` (Proof-of-Origin
Token) query parameter on many videos — that token is generated by YouTube's
`base.js` at fetch time and isn't in the static page state. The extension
reads `performance.getEntriesByType("resource")` to recover the `pot` from
any `/api/timedtext` request YouTube itself has made; if none exists yet,
it programmatically clicks YouTube's CC button twice (toggle on, toggle off)
to trigger a fresh fetch and reads the new URL from the resource-timings
buffer. The popup then formats the data and writes to the clipboard.

See `docs/superpowers/specs/2026-06-07-yt-transcript-copier-design.md` and
`docs/superpowers/specs/2026-06-07-pot-acquisition-design.md` for the full
design.

## Manual end-to-end test checklist

Use this before tagging a release.

- [ ] Load unpacked extension in `chrome://extensions`
- [ ] Open a video with English transcript and **CC off** (e.g. `https://www.youtube.com/watch?v=jNQXAC9IVRw`)
      → icon turns red
- [ ] Click the icon → popup opens with language dropdown + three format buttons
- [ ] Click "Plain text". First copy completes in under ~2 seconds. CC overlay
      does not flash visibly. Paste into a text editor — output is the real
      transcript.
- [ ] Click any format button again on the same video — completion is near-instant
      (in-memory cache hit).
- [ ] Manually enable CC on a video before opening the popup. First click
      completes near-instantly (passive resource-timings scan hits an existing
      entry — no CC button click needed).
- [ ] Switch the language dropdown to a different language and copy. The copied
      content is in that language.
- [ ] SPA-navigate from one transcript-having video to another via an in-page
      link. First copy on the new video re-triggers the CC click. Result is for
      the new video.
- [ ] Open a video with no transcript. Icon stays gray; popup shows
      "This video has no transcript."
- [ ] Open the YouTube homepage or a channel page → icon is gray; popup shows
      "Open a YouTube video to use this."
- [ ] In DevTools Network panel, the outgoing `/api/timedtext` request our
      extension makes includes `pot=...` and `c=WEB`, and the response is
      200 with a non-empty body.
- [ ] Refresh a video page mid-session. The popup still works correctly.
